## Plan: Create OpenSpec change proposal for file-tree naming input + directory delete fix

### Context
Two defects in the Files sidebar right-click context menu:
1. **Directory delete bug** (`src/storage/file_tree.rs:128`): `FileTree::delete` uses `fs::remove_dir`, which only removes *empty* directories. Every directory shown in the tree is necessarily non-empty (it contains at least one Markdown file, else `collect_file_tree_entries` prunes it). So Delete on any folder always fails with "directory not empty".
2. **Missing naming input**: New File / New Folder / Rename all apply a hard-coded default name (`untitled.md`, `New Folder`, `renamed.<ext>` / `Renamed Folder`) with no way for the user to type a name. Rename is effectively broken for its purpose.

### Approach (confirmed with user)
- **Naming input**: in-app inline prompt reusing the existing *redirected text-input* pattern (same mechanism as the search field / file-tree filter). Add a pending-name state; Enter confirms, Escape cancels.
- **Directory delete**: switch to `fs::remove_dir_all` with an extra confirm step for non-empty folders.
